## 1. 문서 개요

### 1-1. 목적
사용자가 디스코드에 올린 일정 정보(텍스트·카카오톡 캡처·사진)를 Claude CLI로 인식·구조화하여
구글 캘린더에 자동 등록하는 개인용 에이전트의 요구사항을 정의한다.

### 1-2. 범위
- 포함: 디스코드 입력 감지, Claude CLI 기반 인식·추출, 구글 캘린더 자동 등록, 디스코드 결과 회신·수정
- 제외(MVP): 반복 일정, 멀티유저, 별도 OCR 엔진, 별도 리마인더

### 1-3. 용어
| 용어 | 정의 |
|------|------|
| 입력 | 디스코드 채널에 올라온 텍스트 또는 이미지 메시지 |
| 추출 | Claude CLI가 입력에서 일정 정보를 구조화 JSON으로 뽑아내는 과정 |
| 일정 객체 | 제목·시작·종료·장소·설명 등으로 구성된 단일 일정 단위 |
| 봇 | 디스코드에서 동작하는 Python 프로세스 |

## 3. 시스템 개요 (처리 흐름)

```
[0] (봇 시작 시) 마지막 처리 시점 이후 미처리 메시지 캐치업
        ↓
[1] 사용자가 외부에서 일정 정보 수신 (문자·카톡·안내문 사진)
        ↓
[2] 사용자가 디스코드 전용 채널에 업로드 (텍스트 또는 이미지 첨부)
        ↓
[3] 봇이 신규 메시지 감지
        ↓
[4] 봇이 입력을 Claude CLI(headless)에 전달  ──── 추출 호출 ①
        ↓
[5] Claude CLI: (이미지면) 비전 OCR → 일정 구조화 추출 (JSON, 다중 가능)
        ↓
   [날짜 불확정 · 이미지 내 상대 날짜] → 디스코드로 되묻기 → 종료
        ↓
[6] 봇이 추출 결과 검토 → Calendar MCP로 등록  ──── 등록 호출 ②
        ↓
[7] 봇이 등록 결과 임베드 회신 ([수정]/[삭제] 버튼, 낮은 신뢰도는 ⚠️ 강조)
        ↓
[8] (선택) 사용자가 버튼·"수정:" 답장으로 사후 보정
```

> 추출(①)과 등록(②)은 **분리된 별도 Claude CLI 호출**이다. 되묻기(FR-13)가 둘 사이에 끼므로,
> 봇이 추출 결과를 받아 판단한 뒤 등록을 진행해야 한다.

## 4. 기능 요구사항 (FR)

우선순위: **M**(Must) / **S**(Should) / **C**(Could) / **W**(Won't, MVP 제외)

### 4-1. 입력 수신

| ID | 요구사항 | 우선 |
|----|----------|------|
| FR-01 | 봇은 지정된 전용 디스코드 채널의 모든 신규 메시지를 자동 감지한다 | M |
| FR-02 | 봇은 텍스트 메시지와 이미지 첨부(단일·복수)를 입력으로 받는다 | M |
| FR-03 | 봇은 텍스트와 이미지가 함께 있는 메시지를 모두 입력으로 처리한다 | S |
| FR-04 | 일정과 무관한 메시지는 추출 0건으로 처리하고 조용히 넘어간다 | C |

### 4-2. 인식 · 추출 (Claude CLI — 추출 호출 ①)

| ID | 요구사항 | 우선 |
|----|----------|------|
| FR-05 | 봇은 입력을 Claude CLI(headless)에 전달해 일정을 구조화 추출한다 | M |
| FR-06 | 이미지 입력은 Claude CLI 비전으로 글자를 인식한다 (별도 OCR 엔진 없음) | M |
| FR-07 | 추출 결과는 정해진 JSON 스키마(§6-1)로 반환한다 | M |
| FR-08 | 한 입력에서 여러 일정을 추출할 수 있다 | M |
| FR-09 | 직접 입력 텍스트(유형 A)의 상대·구어 날짜 표현을 메시지 수신일 기준 절대 날짜로 변환한다. 이미지(유형 B·C) 내 상대 표현은 변환하지 않고 FR-13으로 되묻는다 | M |
| FR-10 | 연도가 없는 날짜는 가장 가까운 미래 연도로 해석한다 | M |
| FR-11 | 시간이 명시되지 않은 일정은 종일(all-day) 일정으로 처리한다 | S |
| FR-12 | 종료 시각이 없으면 기본 소요시간(1시간)을 적용한다 | S |
| FR-13 | 필수 항목(날짜)을 추출할 수 없거나 이미지 내 상대 날짜로 날짜가 불확정이면, 등록하지 않고 디스코드로 되묻는다 | S |

### 4-3. 캘린더 등록 (Claude CLI + MCP — 등록 호출 ②)

| ID | 요구사항 | 우선 |
|----|----------|------|
| FR-14 | 추출 결과 검토 후, Claude CLI + Google Calendar MCP로 기본 캘린더에 자동 등록한다 | M |
| FR-15 | 모든 일정의 타임존은 KST(Asia/Seoul)로 고정한다 | M |
| FR-16 | 동일 제목 + 동일 시작시각 일정이 이미 있으면 중복으로 보고 재등록하지 않는다 | S |
| FR-17 | 반복 일정(RRULE)은 지원하지 않는다 | W |

### 4-4. 결과 회신 · 수정

| ID | 요구사항 | 우선 |
|----|----------|------|
| FR-18 | 등록 후 봇은 결과(제목·일시·장소·캘린더 링크)를 디스코드 임베드로 회신한다 | M |
| FR-19 | 결과 임베드에 [수정]·[삭제] 버튼을 제공한다 | M |
| FR-20 | 사용자가 결과 임베드에 "수정: ..." 형식으로 답장(reply)하면, 봇이 그 임베드에 연결된 캘린더 이벤트를 갱신한다 | M |
| FR-21 | [삭제] 버튼으로 잘못 등록된 이벤트를 캘린더에서 삭제할 수 있다 | M |
| FR-22 | 다중 일정 등록 시 각 일정별로 별도 결과 임베드·버튼을 제공한다 | S |
| FR-23 | 신뢰도(confidence)가 임계값 미만인 일정은 자동 등록하되, 결과 임베드에 "⚠️ 확인 필요"로 강조 표시한다 | S |
| FR-24 | 인식·등록 실패 시 실패 사유를 디스코드로 안내한다 | S |

### 4-5. 운영

| ID | 요구사항 | 우선 |
|----|----------|------|
| FR-25 | 봇은 로컬 맥에서 상시 실행된다 (launchd 등 자동 구동) | M |
| FR-26 | 봇 시작 시 마지막 처리 시점 이후의 미처리 메시지를 따라잡아 처리한다 (다운타임 캐치업) | S |
| FR-27 | 처리 이력·에러를 로그 파일로 기록한다 | S |

## 5. 입력 유형별 처리 요구사항

| 유형 | 설명 | 처리 방식 | 한계·리스크 |
|------|------|-----------|-------------|
| A. 텍스트 | 문자·메시지 붙여넣기 | 원문을 그대로 Claude CLI에 전달. 상대 날짜는 수신일 기준 변환 | 모호한 구어 표현 → 되묻기 |
| B. 카카오톡 캡처 | 대화 캡처 이미지 | 비전으로 말풍선에서 일정 메시지 식별·추출 | 캡처 화질, 말풍선 순서·발신자 혼동, **상대 날짜 → 되묻기** |
| C. 안내문·포스터·손글씨 | 일정 안내 사진 | 비전 OCR 후 추출 | 손글씨 정확도, 복잡한 레이아웃, **상대 날짜 → 되묻기** |

- 모든 유형 공통: 신뢰도가 낮거나 날짜 불확정 시 FR-13(되묻기) / FR-23(강조) 적용.

## 6. 일정 추출 명세

### 6-1. 추출 항목 (JSON 스키마)

| 항목 | 필수 | 설명 |
|------|------|------|
| title | 필수 | 일정 제목 |
| start | 필수 | 시작 일시 (종일이면 날짜만) |
| end | 선택 | 종료 일시 (없으면 FR-12 규칙) |
| all_day | 선택 | 종일 일정 여부 (기본 false) |
| location | 선택 | 장소 |
| description | 선택 | 원문·부가 설명 |
| confidence | 필수 | 추출 신뢰도 (0~1) — 임계값 미만 시 FR-23 강조 |

### 6-2. 날짜 · 시간 해석 규칙

| 상황 | 규칙 |
|------|------|
| 기준일 | 디스코드 메시지 수신 일시 (KST) |
| 상대 표현 (유형 A 텍스트) | "내일/모레/다음 주 화" → 기준일 기준 절대 날짜로 변환 |
| 상대 표현 (유형 B·C 이미지) | 변환하지 않음 — 등록하지 않고 되묻기 (캡처·전달 대화는 수신일 기준이 부정확) |
| 연도 없음 | 가장 가까운 미래 연도로 해석 (예: 5/20 기준 "3/14" → 내년) |
| 시간 없음 | 종일 일정으로 처리 (FR-11) |
| 종료 없음 | 시작 + 1시간 (종일이면 당일 종료) (FR-12) |
| 날짜 자체 불명확 | 등록하지 않고 되묻기 (FR-13) |

### 6-3. 다중 일정
- 한 입력에 여러 일정이 있으면 일정 객체 배열로 추출하고, 각각 개별 등록·개별 임베드 회신한다 (FR-08, FR-22).

## 9. MVP 범위 요약

**MVP 포함 (Must 전체 + 핵심 Should)**
- 전용 채널 감시 → 텍스트·이미지 입력 수신
- Claude CLI 비전 기반 인식·다중 일정 추출
- 상대 날짜 변환(텍스트) / 이미지 상대날짜 되묻기
- 종일 일정 처리, 신뢰도 낮은 일정 ⚠️ 강조
- Calendar MCP 기본 캘린더 자동 등록 (KST)
- 결과 임베드 회신 + [수정]/[삭제] 버튼 + "수정:" 답장 보정
- 로컬 맥 상시 실행 + 다운타임 캐치업

**MVP 제외 (추후)**
- 반복 일정(RRULE)
- 별도 OCR 엔진 폴백
- 별도 리마인더, 멀티유저, 클라우드 배포
